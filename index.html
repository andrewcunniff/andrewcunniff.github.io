What does it take to win games in the NBA?¶

Identifying what stats can predict win percentage.¶

Andrew Cunniff, Tanmay Dhal¶

Introduction¶

The NBA (National Basketball Association) is known to be the best basketball league in the world. In the current 2019-2020 season, there have been a total of 370 total games played, each team with around 25 games under their belt. Since an NBA season consists of 82 games, we are currently only 30% into the season, not including playoffs. You would think that a team’s record is all that’s needed to predict the winner of any game, but that is usually not the case. There are so many different factors that play a major part in winning, so a record doesn’t mean too much when predicting the winning team. Since there are so many different variables in every game, we aim to determine which variables have the strongest prediction power on predicting a team's record. Our end goal for this tutorial is to develop a multiple regression equation to accurately predict a team's winning percentage at the end of the regular season.

1.2 Dataset Source¶

All of our data comes from www.basketball-reference.com. This website has stats for everything basketball related, and provides the ability to convert any table into .csv format. Due to this feature, we did not directly scrape the data ourselves. The data that we targeted includes: team data per 100 possessions and team standings data for the 2018-2019 and 2019-2020 seasons. Since an average game consists of around 100 possessions per team, we thought using data standardized by possession rather than by game would be more useful. That team data consists of variables like points, field goal percentage, turnovers, etc. The standings data has each team's overall record, home and road record, record by month, etc. Here are the links to the data for each season:

1.4 Cleaning Data¶

As you can see, the standings data is a bit in depth, so we'll just take the overall records, convert them to winning percentage, and add it as a column to the team stats dataframe. We created a simple function (winpct) to convert from "W-L" to a winning percent of type float. Another problem we have is that the order of teams is different for the various data sets. We want the order of the overall win% to be in the same order as the team stats, so we will reindex the standings data then transfer the win% data over. There is still more cleaning to be done, but we want to visualize the data before we make any changes.

def winpct(rec):
    wins = float(rec[:rec.index('-')])
    tot = float(rec[rec.index('-')+1:]) + wins
    if tot != 0:
        return wins/tot 
    else:
        return -1.0

def monthly_record(df):
    newoct = []
    newnov = []
    newdec = []
    for idx,row in df.iterrows():
        octrec = row['Oct']
        octw = float(octrec[:octrec.index('-')])
        octt = float(octrec[octrec.index('-')+1:]) + octw
        newoct.append(octw/octt)
        
        novrec = row['Nov']
        novw = float(novrec[:novrec.index('-')])
        novt = float(novrec[novrec.index('-')+1:]) + novw
        newnov.append((novw+octw)/(novt+octt))
        
        decrec = row['Dec']
        decw = float(decrec[:decrec.index('-')])
        dect = float(decrec[decrec.index('-')+1:]) + decw
        newdec.append((decw+novw+octw)/(dect+novt+octt))
    return pd.DataFrame({'Team':df.Team,9:[0]*30,10:newoct,11:newnov,12:newdec})

4. Reveal key stats with Multiple Linear Regression¶

4.1 Null Hypothesis¶

Null Hypothesis: None of the variables have a significant impact on winning percentage.¶

We will test this null hypothesis with joint hypothesis testing using the F-statistic.

4.2 Regression¶

In order to find the best possible model, we will use the best subset selection algorithm. This essentially checks the Residual Sum of Squares (RSS) and R-Squared for every possible combination of variables that you feed it. After it finishes, we can identify the combination of variables that result in the lowest RSS or highest R-Squared for any number of variables. To start off, We'll use the variables that have a correlation with ovr over .3. These are points, field goals, defensive rebounds, field goal %, 2-pointer %, total rebounds, 3-pointer %, assists, blocks, 3-pointers made, 3-pointers attempted, and turnovers.

  • If you would like to learn more about the thoery behind subset selection, check out chapter 6 in this statistical learning textbook: http://faculty.marshall.usc.edu/gareth-james/ISL/ISLR%20Seventh%20Printing.pdf.
  • If you would like to learn more about the coding behind Python Regression analysis, check out the series of articles on https://medium.com/@dhwajraj.
  • If you need help with coding subset selection algorithms, this notebook helps: http://www.science.smith.edu/~jcrouser/SDS293/labs/lab8-py.html.

def fit_linear_reg(X,Y):
    #Fit linear regression model and return RSS and R squared values
    model_k = linear_model.LinearRegression(fit_intercept = True)
    model_k.fit(X,Y)
    RSS = mean_squared_error(Y,model_k.predict(X)) * len(Y)
    R_squared = model_k.score(X,Y)
    return RSS, R_squared

def subset_selection(df,y):
    Y = df[y]
    X = df.drop(y, axis = 1)
    k = len(df.columns)-1
    RSS_list, R_squared_list, vars_list, num_vars = [],[],[],[]

    #Looping over k = 1 to k = 11 features in X
    for k in tnrange(1,len(X.columns) + 1, desc = 'Loop...'):

        #Looping over all possible combinations
        for combo in itertools.combinations(X.columns,k):
            temp = fit_linear_reg(X[list(combo)],Y)   #Store temp result 
            RSS_list.append(temp[0])                  #Append lists
            R_squared_list.append(temp[1])
            vars_list.append(combo)
            num_vars.append(len(combo))   

    #Store in DataFrame
    rdf = pd.DataFrame({'num_vars': num_vars,'RSS': RSS_list, 'R_squared':R_squared_list,'vars':vars_list})
    df_max = rdf[rdf.groupby('num_vars')['R_squared'].transform(max) == rdf['R_squared']]
    display(df_max.head(k))
    return df_max
